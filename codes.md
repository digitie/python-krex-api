라이브러리에서는 모두 `kex_openapi.codes` 모듈에 enum 형태로 제공됩니다.

> **주의**
> 일부 코드(특히 영업소 코드)는 도로 신설/변경에 따라 추가·수정됩니다.
> 운영 시점의 정확한 값은 `client.tollfee.tollgate_list()` 또는
> `client.reference.routes()`로 동기화하세요.

## 차종 (CarType)

통행료 산정과 교통량 집계의 기준 분류.

| 코드 | Enum | 한글명 | 차량 분류 기준 |
|------|------|--------|----------------|
| `1` | `CarType.LIGHT` | 1종 | 2축 차량 중 윤폭 279.4mm 이하<br>(승용차, 16인승 이하 승합, 2.5톤 이하 화물) |
| `2` | `CarType.MEDIUM` | 2종 | 2축 차량 중 윤폭 279.4mm 초과<br>(2.5톤 초과 ~ 7톤 미만, 16인승 초과 32인승 이하 승합) |
| `3` | `CarType.LARGE_3AXLE` | 3종 | 3축 차량<br>(7톤 이상 화물 + 32인승 초과 버스) |
| `4` | `CarType.LARGE_4AXLE` | 4종 | 4축 차량<br>(10톤 이상 대형 화물) |
| `5` | `CarType.LARGE_5AXLE` | 5종 | 5축 이상 특수 차량 |
| `6` | `CarType.LIGHT_DISCOUNT` | 6종 | 1종 중 경형(1,000cc 미만) — 50% 할인 대상 |

```python
from kex_openapi.codes import CarType
CarType.LIGHT.value          # "1"
CarType.LIGHT.label          # "1종 (소형)"
CarType.from_label("1종")     # CarType.LIGHT
```

## TCS / 하이패스 (TCSType)

요금 수납 방식 구분.

| 코드 | Enum | 설명 |
|------|------|------|
| `1` | `TCSType.TCS` | TCS — 영업소 현장 수납 (현금/카드/단말 미장착) |
| `2` | `TCSType.HIPASS` | 하이패스 — 무정차 통과 |
| `0` | `TCSType.ALL` | TCS + 하이패스 합계 *(일부 엔드포인트만 지원)* |

> 일부 엔드포인트는 `tcsType`이 필수이며 `0`을 받지 않습니다.
> 합계가 필요하면 `1`과 `2`를 따로 호출 후 합산하세요.

## 도로 운영기관 (RoadOperator)

`exDivCode` 파라미터에 사용. 한국도로공사가 직접 운영하는 노선과 민자 노선을 구분합니다.

| 코드 | Enum | 운영기관 | 대표 노선 |
|------|------|----------|-----------|
| `00` | `RoadOperator.KEC` | 한국도로공사 | 경부선, 서해안선, 호남선 등 대부분 |
| `01` | `RoadOperator.PRIV_INCHEON_AIRPORT` | 신공항하이웨이(주) | 인천공항고속도로 |
| `02` | `RoadOperator.PRIV_CHEONAN_NONSAN` | 천안논산고속도로(주) | 천안논산고속도로 |
| `08` | `RoadOperator.PRIV_DAEGU_BUSAN` | 대구부산고속도로(주) | 대구부산고속도로 |
| `11` | `RoadOperator.PRIV_SEOUL_CHUNCHEON` | 서울춘천고속도로(주) | 서울춘천고속도로 |
| `18` | `RoadOperator.PRIV_GURIPOCHEON` | 서울북부고속도로(주) | 구리포천고속도로 |

> 민자 노선이 추가되거나 운영주체가 변경되면 코드도 추가됩니다.
> 위 표는 라이브러리 v1.0 기준이며, 알려지지 않은 코드를 만나면
> `RoadOperator.OTHER` (또는 raw 문자열)로 폴백합니다.

## 혼잡도 (CongestionLevel)

실시간 소통 정보의 혼잡 수준.

| 코드 | Enum | 색상 | 통행속도 기준 (대략) |
|------|------|------|----------------------|
| `1` | `CongestionLevel.SMOOTH` | 🟢 녹색 | ≥ 80 km/h |
| `2` | `CongestionLevel.SLOW` | 🟡 황색 | 40 ~ 80 km/h |
| `3` | `CongestionLevel.DELAY` | 🟠 주황 | 20 ~ 40 km/h |
| `4` | `CongestionLevel.STOP` | 🔴 적색 | < 20 km/h |

기준 속도는 노선마다 다르며, `reference.routes()`의
`freeFlowSpeed` / `congestionSpeed`로 확인할 수 있습니다.

### 코드 검색

라이브러리에는 명칭→코드 양방향 검색 헬퍼가 포함됩니다.

```python
from kex_openapi import KexClient
client = KexClient()

# 명칭으로 검색
matches = client.codes.tollgate_lookup("부산")
# [TollgateInfo(unit_code="551", name="부산", route="경부선"), ...]

# 코드로 조회
info = client.codes.tollgate_get("101")
# TollgateInfo(unit_code="101", name="...", ...)

# 노선 + 명칭으로 정확 매치
info = client.codes.tollgate_lookup("동대구", route="경부선", exact=True)
```

내부 코드 캐시는 24시간마다 자동 갱신되며, 강제 새로고침은
`client.codes.refresh()`로 수행합니다.

## 코드값을 안전하게 다루는 패턴

라이브러리는 enum과 raw 문자열 모두 받습니다.

```python
# 둘 다 동일하게 동작
client.traffic.by_ic(car_type=CarType.LIGHT, ...)
client.traffic.by_ic(car_type="1", ...)
client.traffic.by_ic(car_type=1, ...)
```

알 수 없는 코드는 `UnknownCode` enum 멤버로 보존되며 `.raw_value`로 원본을 확인할 수 있습니다.

```python
result = client.traffic.flow()
for item in result.items:
    if isinstance(item.car_type, UnknownCode):
        log.warning(f"새 차종 코드 발견: {item.car_type.raw_value}")
```
